# Лабораторна робота №1

## Тема

**Створення REST сервісу на основі специфікації OpenAPI**

## Мета роботи

Метою лабораторної роботи є отримання практичних навичок створення REST API на основі специфікації OpenAPI, генерації серверної частини за допомогою Swagger Codegen та реалізації логіки обробки HTTP-запитів у Spring Boot.

## Варіант завдання

**API Бюро з працевлаштування**

Сервіс повинен надавати можливість працювати з такими об’єктами предметної області:

### Роботодавці

- назва;
- вид діяльності;
- адреса;
- телефон.

### Вакансії

- роботодавець;
- найменування посади;
- передбачуваний розмір заробітної плати;
- перелік вимог;
- соціальні пільги.

---

## Хід виконання роботи

### 1. Аналіз предметної області

На першому етапі було проаналізовано предметну область «Бюро з працевлаштування». Для реалізації REST-сервісу було визначено дві основні сутності:

- `Employer` — роботодавець;
- `Vacancy` — вакансія.

Для кожної сутності було передбачено виконання базових CRUD-операцій:

- створення запису;
- отримання списку записів;
- отримання одного запису за ідентифікатором;
- оновлення запису;
- видалення запису.

---

### 2. Проєктування REST API

Для роботи з роботодавцями було визначено такі кінцеві точки API:

| Метод | Endpoint | Призначення |
|---|---|---|
| GET | `/employers` | Отримання списку всіх роботодавців |
| GET | `/employers/{employer_id}` | Отримання інформації про конкретного роботодавця |
| POST | `/employers` | Додавання нового роботодавця |
| PUT | `/employers/{employer_id}` | Оновлення даних роботодавця |
| DELETE | `/employers/{employer_id}` | Видалення роботодавця |

Для роботи з вакансіями було визначено такі кінцеві точки API:

| Метод | Endpoint | Призначення |
|---|---|---|
| GET | `/vacancies` | Отримання списку всіх вакансій |
| GET | `/vacancies/{vacancy_id}` | Отримання інформації про конкретну вакансію |
| POST | `/vacancies` | Додавання нової вакансії |
| PUT | `/vacancies/{vacancy_id}` | Оновлення даних вакансії |
| DELETE | `/vacancies/{vacancy_id}` | Видалення вакансії |

---

### 3. Створення специфікації OpenAPI

Після визначення структури REST API було створено декларативну специфікацію OpenAPI у форматі YAML.

У специфікації було описано:

- доступні маршрути API;
- HTTP-методи для кожного маршруту;
- параметри запитів;
- структуру JSON-об’єктів для запитів `POST` та `PUT`;
- формати відповідей сервера;
- повідомлення про можливі помилки.

Використання OpenAPI дозволило стандартизувати опис REST-сервісу та спростити подальшу генерацію серверного коду.

---

### 4. Генерація серверної частини

На основі створеної OpenAPI-специфікації було згенеровано серверну частину API за допомогою інструменту **Swagger Codegen**.

Для реалізації було використано:

- мову програмування **Java**;
- фреймворк **Spring Boot**;
- систему збирання **Maven**;
- середовище розробки **Visual Studio Code**.

Після генерації проєкту було отримано базову структуру серверного застосунку, яка містила моделі, інтерфейси API та контролери.

---

### 5. Реалізація логіки контролерів

Оскільки згенерований код містив лише шаблонну структуру, було реалізовано логіку обробки запитів у контролерах:

- `EmployersApiController`;
- `VacanciesApiController`.

Для тимчасового зберігання даних було використано колекції `ArrayList`, тобто дані зберігаються в оперативній пам’яті під час роботи сервера.

Приклад реалізації сховища для вакансій:

```java
private static final List<Vacancy> vacancies = new ArrayList<>();
```

У контролерах було реалізовано методи для:

- отримання списку об’єктів;
- створення нового об’єкта;
- автоматичного встановлення ідентифікатора;
- повернення відповідних HTTP-статусів.

Наприклад, під час створення нової вакансії сервер повертає статус:

```java
HttpStatus.CREATED
```

А під час успішного отримання списку вакансій повертається статус:

```java
HttpStatus.OK
```

---

### 6. Запуск проєкту

Після реалізації серверної логіки проєкт було зібрано та запущено локально.

Для запуску Spring Boot-застосунку використовувалась система Maven. Після старту сервера REST API став доступним для тестування через локальний порт.

---

### 7. Тестування API

Для перевірки працездатності REST API було використано інтерфейс **Swagger UI**.

За допомогою Swagger UI було протестовано роботу основних HTTP-запитів:

- `POST` — створення нового роботодавця або вакансії;
- `GET` — отримання списку записів;
- `PUT` — оновлення існуючих даних;
- `DELETE` — видалення записів.

Під час тестування було виконано POST-запит для створення нової вакансії. Сервер успішно обробив запит та повернув відповідь зі статусом:

```text
201 Created
```

Після цього було виконано GET-запит для отримання списку вакансій. У відповіді сервер повернув JSON-масив з доданими об’єктами та статус:

```text
200 OK
```

Це підтвердило правильність роботи реалізованого REST-сервісу.

## Висновок

У ході виконання лабораторної роботи було створено REST-сервіс для предметної області «Бюро з працевлаштування» на основі специфікації OpenAPI.

Було розроблено структуру API для роботи з роботодавцями та вакансіями, створено OpenAPI-специфікацію, згенеровано серверну частину за допомогою Swagger Codegen та реалізовано логіку контролерів у Spring Boot.
